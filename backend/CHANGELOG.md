# Changelog
Все значимые изменения должны записываться сюда в формате [версия сервиса] - дата изменения - тип изменения (Fixed, Changed, Added) и описание изменения

### Пример:
# [2.2.8] - 2022-01-09
### Added
* Добавление документации в сервис с полным описанием работы
* Изменение сервисной логики ретрая приложения при панике
